Image-browser: Use the API

Querying images

Image-browser uses imgur's API to query images. This means our images are external, so there's an additional HTTP request in the mix. Rather than
Client(you) <---> Our Server
There is a "third party":
Client(you) <---> Our Server <---> Imgur Server    <---Not optimal
This means we have no control over image compression, what images you'll see, or what usage license(s) are attributed to them.
So, it's quite obvious that this service is unoptimized and shouldn't be used in production! It's really meant as a learning exercise both for me and you. So, feel free to fiddle around with it.
If you still want to use this service, here is how:

GET Request

Note that if you run this on your local machine, replace the url from start to "/api/search?..." with your domain name or http://localhost:port.

GET ---> https://image-browser.glitch.me/api/search?query

Troubleshooting

    Limits

    Imgur puts a 12,500 reqests/day limit on our API.
    Also, Users (via IP Address) that repeatedly sends requests will be blocked by Imgur. See the docs for more info
